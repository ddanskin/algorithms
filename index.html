Bubble Sort

  • Bubble sort has worst case and average case complexity O(n^2)
  • It is not practical when N is large
  • Best case complexity is O(n), when list is already sorted

Insertion Sort

  • Insertion sort has worst case and average case complexity O(n^2)
  • Works best with small data sets or data sets that are already mostly sorted
  • Best case performance when list is mostly sorted is O(n) comparisons, O(1) swaps

Selection Sort

  • Selection sort has worst case and average case complexity 0(n^2)
  • It performs poorly on large data sets and it generally performs worse than insertion sort
  • Best case performance is O(n^2), but it can have performance advantage over other more complicated algorithms when auxiliary memory is limited

Merge Sort

  • Merge sort's worst case and average case complexity is 0(n log n)
  • Typical best case complexity is 0(n log n), but natural variant is 0(n)
